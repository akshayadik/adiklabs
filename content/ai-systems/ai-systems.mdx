## 1. Problem Statement

Modern systems must protect themselves from:
- Traffic spikes
- Abuse (bots, brute force)
- Resource exhaustion

A **Rate Limiter** ensures that a client can only make a limited number of requests within a defined time window.

---

## 2. Requirements

### Functional
- Limit requests per user / API key / IP
- Configurable limits (e.g., 100 requests/min)
- Support multiple APIs

### Non-Functional
- Low latency (< 5ms overhead)
- Highly scalable (millions of users)
- Strong consistency (no over-allowing)
- Fault tolerant
